# 书库

书库不要用一套主题表粗暴覆盖所有书。每个学科有自己的内部秩序。

### 根目录结构

如果用户使用 Calibre，不要手动整理 `CalibreLibrary` 内部文件。Calibre 数据库自己负责物理层。

手动书库根目录先按系统角色分层：

```
EbookLibrary/
├── CalibreLibrary/
├── 藏书/
├── 待整理/
├── 下载原包/
├── 编辑工作区/
└── 回收/
```

`藏书/` 内部再按学科分大类：

```
藏书/
├── 哲学与思想/
├── 文学与写作/
├── 历史地理/
├── 电影与艺术/
├── 音乐与声音/
├── 社会政治/
├── 心理与生命经验/
├── 医学与身体/
├── 科学与自然/
├── 技术与计算/
├── 宗教与灵性/
└── 工具书与资料/
```

### 学科内部主轴

| 学科 | 最适合的分类主轴 | 为什么 |
|---|---|---|
| 哲学 | 思想传统 + 历史谱系 | 哲学强烈依赖传承、反驳、转向 |
| 文学 | 语言/地域传统 + 作者/时期 | 文学首先依赖语言和作品传统 |
| 历史 | 地域 + 时间 | 历史天然是空间和时间的交叉 |
| 艺术 | 媒介 + 艺术史阶段/流派 | 绘画、建筑、电影、摄影的媒介差异很大 |
| 电影理论 | 电影史阶段 + 理论问题/作者 | 既有媒介史，也有理论传统 |
| 音乐 | 类型/传统 + 作曲家/时代 | 古典、爵士、流行、实验音乐不是同一套逻辑 |
| 社会科学 | 学科对象 + 方法 | 社会学、政治学、人类学、经济学看问题方式不同 |
| 政治法律 | 制度领域 + 法域/理论传统 | 法律尤其依赖国家/法域 |
| 心理学 | 理论流派 + 应用场景 | 精神分析、认知、创伤、咨询等路径不同 |
| 医学 | 身体系统 + 临床用途 | 医学是实践性知识，按身体/疾病/治疗更稳 |
| 科学 | 自然对象层级 | 数学、物理、化学、生物等有明确对象层级 |
| 技术计算 | 功能/技术栈/应用场景 | 技术变化快，按历史分期没用 |
| 宗教 | 宗教传统 + 教派/实践 | 佛教、基督教、伊斯兰教、道教各有内部传统 |
| 工具书 | 用途 | 字典、手册、参考资料就是查用工具 |

### 书库规则

- 不使用数字前缀作为默认排序手段。
- 根目录主轴是系统角色；`藏书/` 内部主轴是学科族群。
- 不把下载来源（Anna/Z-Library）和主题目录放在同一层；来源包进 `下载原包/` 或 `待整理/`。
- 不把阅读状态（待读/已读/备用/回收）和学科目录放在同一层。
- 收藏型书籍使用原书名，不加日期。
- 跨学科书放最主要归属，其他归属交给标签、读书笔记或 MOC。
- 临时研究 PDF 先进 `待整理/`，定期入库或删除。

### 书库验证

```bash
# 根目录应是系统角色，而不是主题、来源、状态混杂
find EbookLibrary -mindepth 1 -maxdepth 1 -type d -print

# 根目录不应散落书籍文件
find EbookLibrary -mindepth 1 -maxdepth 1 -type f ! -name ".DS_Store" -print

# 藏书大类应按学科族群
find EbookLibrary/藏书 -mindepth 1 -maxdepth 1 -type d -print

# 藏书内部不应过深
find EbookLibrary/藏书 -mindepth 5 -type d -print | head -5
```
