# 文件夹

通用文件夹先判断“系统角色”，再决定是否需要主题分类。

| 文件夹角色 | 用途 | 规则 |
|---|---|---|
| 待整理 | 新进入、未判断 | 定期清空 |
| 藏品/资料库 | 长期保存 | 使用稳定主轴 |
| 项目 | 服务明确交付物 | 项目边界优先 |
| 工作区 | 正在编辑、转换、处理 | 结束后归档或删除 |
| 来源备份 | 保留下载原包 | 不和主题目录混放 |
| 回收 | 暂缓删除 | 定期清空 |

通用结构：

```
Assets/
├── 藏品/
├── 待整理/
├── 项目/
├── 工作区/
├── 来源备份/
└── 回收/
```

规则：

- 根目录按系统角色分，不按主题堆。
- 项目文件夹内可以按交付结构分：`Refs/`、`Assets/`、`Exports/`、`Archive/`。
- 收藏型资料进入 `藏品/` 后，再按资产类型或学科方法细分。
- 不建“重要”“有用”“以后看”“喜欢”这类情绪目录。
