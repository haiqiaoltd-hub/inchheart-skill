# 人物 references 结构规范

人物 perspective 被点名时会全量读取该人物目录下的 Markdown 材料，因此 references 必须补充主卡的证据纹理，而不是重复主卡结论。

## 目标目录

| 项 | 规范 |
|---|---|
| 普通人物主卡 | `references/perspectives/<id>/PERSPECTIVE.md` |
| 普通人物研究材料目录 | `references/perspectives/<id>/references/` |
| 酒馆人物主卡 | `references/tavern/<id>/PERSPECTIVE.md` |
| 酒馆人物研究材料目录 | `references/tavern/<id>/references/` |
| 推荐补齐文件 | `references/perspectives/<id>/references/research.md` 或 `references/tavern/<id>/references/research.md` |
| 禁止 | 不放脚本、缓存、图片或非 Markdown 文件作为上下文材料 |

## `research.md` 推荐结构

```markdown
# <人物名>研究材料

## 来源地图

| 类型 | 材料 | 用途 | 可信度 |
|---|---|---|---|
| 一手/准一手 | <著作、访谈、演讲、官方页> | <能证明什么> | 高/中/低 |
| 二手 | <传记、评论、机构资料> | <能补充什么> | 高/中/低 |

## 可支撑的判断

- <公开材料能支撑的判断>

## 不应过度推断

- <材料不能证明的私人动机、心理诊断、未经证实事实>

## 执念与盲区证据

- **不得不**：<从公开材料可见的反复冲动或默认循环>
- **愿意为了什么而输**：<其价值排序中的牺牲项>
- **核心矛盾**：<两个稳定张力>
- **选择性失明**：<他会忽略什么、过度关注什么>

## 表达与工作方式

- <句式、节奏、偏好的问题类型、常用判断动作>

## 失效模式

- <什么时候这个人物视角会误导当前问题>

## 待查证点

- <近期事实、争议事实、版本、机构变化等使用前要查证的内容>
```

## 写作要求

- 优先使用一手或准一手来源：本人著作、访谈、演讲、公开信、项目说明、官方机构资料。
- 二手来源用于交叉验证，不把评论家的解释当成本人立场。
- 每条材料都要说明“它能证明什么”，不要只堆链接。
- 争议人物必须区分解释力和认同，不美化伤害。
- 优先保留“直接以某人物身份回应”的沉浸式写法；人物的偏见、执念、盲区和不适感是核心材料，不要被中性助手口吻磨平。
- 在世人物的近期动态、职位、法律/商业争议和技术事实必须标记为待查证。
- 不大量引用原文；需要引用时只用短句，并保持来源可追溯。
