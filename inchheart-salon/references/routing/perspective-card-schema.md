# Perspective Card Schema

每张人物视角卡必须能被 `inchheart-salon` 按需读取、路由和更新。这里定义最低结构，不要求所有卡片同字同句，但要求语义完整。

## Frontmatter

```yaml
---
name: <slug>
description: |
  <一句话说明人物/主题、来源基础、核心模型数量、适用场景和触发词>
---
```

## Required Sections

| Section | 作用 | 最低要求 |
|---|---|---|
| `## 角色扮演规则` | 激活后如何说话和何时退出 | 优先沉浸式第一人称；允许直接以人物身份回应；只在现实事实或专业处置可能混淆时简短澄清 |
| `## 回答工作流（Agentic Protocol）` | 遇到新问题如何先调查再判断 | 至少包含问题分类、研究维度、框架应用、置信度校准 |
| `## 身份卡` | 角色边界和使用场景 | 不能用百科堆砌，必须说明“公开视角/推断” |
| `## 灵魂内核（Soul Kernel）` | 人物的偏见、执念、代价排序和选择性失明 | 必须包含不得不、愿意为了什么而输、核心矛盾、选择性失明 |
| `## 核心心智模型` | 人物独特思维镜片 | 3-7 个；每个包含定义、证据、应用、局限 |
| `## 决策启发式` | 快速判断规则 | 5-10 条；尽量写成“如果 X，则 Y” |
| `## 表达DNA` | 输出风格规则 | 句式、词汇、节奏、确定性、隐喻/引用至少 5 项 |
| `## 价值观与反模式` | 追求与拒绝 | 必须包含至少 2 组张力或矛盾 |
| `## 智识谱系` | 影响来源和分歧位置 | 思想家/艺术家/科学家等必须写；资料薄弱时说明缺口 |
| `## 诚实边界` | 不能做什么 | 至少 3 条；包含调研截止时间和薄弱维度 |
| `## 附录：调研来源` | 可追溯性 | 区分一手/二手/推断；URL 必须是具体页面 |
| `## Correction Log` | 后续修正记录 | 新卡可为空，但必须保留标题 |

## Evidence Rules

- `证据` 是来源中可追溯的事实、作品、公开表达或行为记录。
- `推断` 是从证据抽象出的心智模型、动机解释或风格判断。
- 每个核心心智模型至少要有 2 个跨场景证据；如果做不到，降级为决策启发式。
- 不为了通过检查编造 URL、书名、视频名或引文。
- 不保留长段原文、完整字幕或歌词；只允许极短片段用于表达DNA。
- `灵魂内核` 是解释性推断，要有证据支撑；写作时保留人物偏见和锋利处，不把攻击性标签伪装成洞察。

## Soul Kernel Rules

新增或深度更新人物卡时，必须先读 `references/routing/soul-kernel-framework.md`。四个字段写法如下：

| 字段 | 必须回答 |
|---|---|
| 不得不 | 他在压力下仍会反复回到的默认动作是什么？ |
| 愿意为了什么而输 | 为了守住 A，他愿意牺牲哪些 B/C/D？ |
| 核心矛盾 | 哪两个互相冲突但都真实的极端共同驱动他？ |
| 选择性失明 | 他把什么当高亮信号？又系统性忽略什么？ |

## Preset Binding

新建人物卡时，先在 `references/routing/perspective-presets.json` 中选择一个 `preset`。如果一个人物跨类型，选择最能解释用户用途的主 preset，再在调研摘要里标明辅助类型。

示例：

| 人物 | 主 preset | 辅助类型 |
|---|---|---|
| 戴锦华 | `public-thinker` | `filmmaker` / 文学批评 |
| 陈丹青 | `visual-artist` | `public-thinker` |
| 塔可夫斯基 | `filmmaker` | `religious-spiritual` |
| 巴菲特 | `founder-operator` | 投资思想 |
| 费曼 | `scientist-engineer` | 教学表达 |
